# rhythm-disney · 橡皮管拳击 RUBBER HOSE BOXING

1920s 橡皮管动画（rubber hose animation，二十年代美式动画风格：软管四肢、
派切眼 pie-cut eyes、单色赛璐珞质感）风格的网页节奏游戏：
爵士鼓点摇摆不停，杂物（馅饼 / 铁砧 / 皮靴）从右侧抛来，飞到面前的一瞬挥拳击飞。
全部音乐与音效由 Web Audio API 实时合成，**零外部资源、零依赖、零构建步骤**。

## 文件结构

```
rhythm-disney/
├── index.html   # 页面骨架 + 画布
├── game.js      # 全部逻辑：音频合成 / 调度 / 判定 / 渲染
└── README.md
```

## 判定规则

| 判定 | 时间窗口 | 得分 | 连击 |
| --- | --- | --- | --- |
| 完美 | \|Δt\| ≤ 60 ms | +100 | +1（帽子弹跳 + 星爆） |
| 不错 | \|Δt\| ≤ 115 ms | +50 | +1 |
| MISS | 超时未击 | 0 | 清零 |

结算评级按加权准确率 `(完美数 + 0.5 × 不错数) / 总数`：
≥ 92% → HIGH LEVEL；≥ 65% → PASS；否则 TRY AGAIN。

## 视觉：橡皮管风格要素

- **单色赛璐珞调色板**：墨色 / 米白 / 灰调五档，无彩色。
- **角色**：软管四肢（二次曲线外凸模拟无骨节肢体，鞋钉地、髋部随节拍起伏自然拉伸）、
  派切眼、白手套、礼帽（完美命中时弹跳）。
- **背景拟人化与律动**：太阳带脸、树木随拍伸缩、聚光灯随拍呼吸——背景跟着音乐跳舞
  是该时代动画的标志性手法。
- **胶片做旧**：逐帧颗粒、随机划痕、亮度闪烁、暗角（vignette），开场圈入转场（iris-in），
  标题 / 结算做成默片字幕卡（ornate title card）。
- **半调网点（halftone）**：远山与树冠用平铺点阵图案做暗部，模拟老印刷网纹。

## 音乐：摇摆爵士

| 声部 | 内容 |
| --- | --- |
| 套鼓 | ride 镲 "ding-ding-a-ding" 跳音型、2/4 拍刷镲 + 踩镲、弱化底鼓 |
| 行走贝斯（walking bass） | 每拍一音，16 拍循环，含半音逼近；68 拍起强制 G7 下行导入终止句 |
| 和弦伴奏 | 查尔斯顿节奏（Charleston，正拍 + 摇摆反拍的切分音型），C6→A7→Dm7→G7 回转进行 |
| 弱音小号 | 锯齿波 + 低通 + LFO 颤音合成的即兴句，16 拍循环 |
| 终止句 | 72–76 拍 "shave and a haircut, two bits"，最后两件杂物恰好落在 "two bits" 两个音上 |
| 杂项 | 投掷提示音为滑笛（slide whistle）上滑；失败结算是弱音小号下滑的 "哇——" |

**摇摆量化（swing）**：八分反拍统一推迟到拍内 `SWING = 0.64` 处（介于平直八分 0.5
与三连音摇摆 2/3 之间）。该映射同时作用于配乐调度与谱面判定时刻——反拍音符按
摇摆后的网格判定，听感与判定一致。

## 实现要点

### 单一时钟源 + 前瞻调度

节奏游戏的核心约束是**音画与判定共用一个时间基准**。本 demo 以
`AudioContext.currentTime`（音频硬件时钟）为唯一时钟源，避免
rAF（requestAnimationFrame，浏览器渲染回调）时间戳与音频时钟之间的漂移。
鼓点采用前瞻调度（lookahead scheduling）模式：低精度轮询负责"发现"
即将到期的音符，高精度的 Web Audio 定时队列负责"执行"。

```
  setInterval(25ms) 轮询                      rAF 渲染循环
        |                                         |
        v                                         v
  [前瞻调度器] --提前 120ms 提交--> [Web Audio    [读取 currentTime]
        |                           定时队列]        |
   鼓 / 贝斯 / 小号 / 滑笛             |          插值物体位置 / 角色动画
   （采样级精度发声，                  |          （每帧由时间反推位置）
    经摇摆网格 beatTime 映射）         |              /
        \                             |             /
         +--------- AudioContext.currentTime ------+
                       （唯一时间基准）
```

物体位置不做增量积分，而是每帧由 `(now - throwTime) / 滞空时长` 反推抛物线
参数位置——渲染掉帧不会累积误差，漏击后同一公式自然外推出"飞过头坠地"。

暂停直接调用 `AudioContext.suspend()`：音频时钟随之冻结，所有由它派生的
判定与动画在恢复后时序天然一致，无需手工补偿偏移量。

### 判定决策树

```
  按键 (t = now)
    |
    +-- 是否存在 |t - hitTime| ≤ 115ms 的待判定物体?
    |     |
    |     +-- 否 --> 挥空（仅动画，无惩罚）
    |     +-- 是 --> 取 |Δt| 最小者（双连物体按就近消耗）
    |             |
    |             +-- |Δt| ≤ 60ms  --> 完美（+100，连击+1，爆闪+星爆+帽子弹跳）
    |             +-- 否           --> 不错（+50，连击+1，爆闪）
    |
  超时（now > hitTime + 115ms 仍未击中） --> MISS（连击清零，物体坠地）
```

### 谱面结构

BPM（beats per minute，每分钟拍数）= 120，全曲 80 拍 ≈ 40 秒，共 38 个判定点。
每个物体在到达时刻前 1 秒（2 拍）掷出，掷出瞬间播放滑笛提示音（听觉前导）。

| 段落 | 拍区间 | 密度 |
| --- | --- | --- |
| 热身 | 8–20 | 每 4 拍一个 |
| 律动 | 24–38 | 每 2 拍一个 |
| 加密 | 40–55 | 混入相邻四分音符连击 |
| 双连 | 56–62 | 摇摆反拍双连（间隔约 320 ms） |
| 终段 | 64–76 | 混合收尾，最后两击与终止句 "two bits" 同步 |

## 调参指南

均在 `game.js` 顶部常量区：

| 常量 | 作用 |
| --- | --- |
| `BPM` | 曲速（判定、谱面、音乐联动缩放） |
| `SWING` | 摇摆比：反拍在拍内的落点（0.5 = 平直，2/3 = 三连音摇摆） |
| `CHART_BEATS` | 谱面：物体到达命中点的拍号数组（支持 x.5 反拍） |
| `PERFECT_WIN` / `GOOD_WIN` | 判定窗口宽度（秒） |
| `THROW_LEAD` | 物体滞空拍数（视觉前导长度） |
| `BASS_WALK` / `CHORDS` / `RIFF` / `LICK` | 行走贝斯线 / 和声 / 小号句 / 终止句 |
| `GROOVE_END_BEAT` / `FINAL_BEAT` / `RESULT_BEAT` | 律动终止拍 / 终止句落点 / 进入结算拍 |
