# rhythm-disney · 橡皮管歌舞秀 RUBBER HOSE REVUE

1920s 橡皮管动画（rubber hose animation，二十年代美式动画风格：软管四肢、
派切眼 pie-cut eyes、单色赛璐珞质感）风格的网页节奏游戏合集，
含四个输入机制各不相同的小游戏（节目单菜单选关）。
全部音乐与音效由 Web Audio API 实时合成，**零外部资源、零依赖、零构建步骤**。

## 文件结构

```
rhythm-disney/
├── index.html       # 页面骨架 + 画布
├── core.js          # 共享引擎：时钟/调度/判定/合成器/胶片效果/字幕卡/菜单
├── game-boxing.js   # 节目一：橡皮管拳击（反应计时）
├── game-tap.js      # 节目二：踢踏回声（呼叫-应答记忆）
├── game-chorus.js   # 节目三：剧院三重唱（长按起止双判定）
├── game-rally.js    # 节目四：橡皮羽球（2 拍 / 4 拍往返对拉）
└── README.md
```

## 运行

任选其一：

```bash
# 方式一：直接用浏览器打开（无跨域资源，file:// 协议即可）
open index.html

# 方式二：本地静态服务器
python3 -m http.server 8000   # 然后访问 http://localhost:8000
```

## 操作

| 按键 | 作用 |
| --- | --- |
| 1-4 或点击（节目单） | 选择节目 |
| 空格 / J / 鼠标（游戏内） | 动作键（三重唱为按住/松开） |
| P | 暂停 / 继续 |
| Esc | 回节目单 |
| 空格（结算卡） | 再演一场 |

按键隔离：带 Cmd / Ctrl / Alt 修饰键的组合一律放行给浏览器（切标签、打印等
快捷键不会触发游戏动作）；游戏消费的裸按键统一 preventDefault，阻断空格滚动
等浏览器默认行为；画布上仅主键点击参与判定，右键菜单被屏蔽；窗口失焦自动
进入幕间休息。

## 四个节目

| 节目 | 输入机制 | BPM | 配乐 |
| --- | --- | --- | --- |
| 橡皮管拳击 | 反应计时：单次按键 | 120 | 摇摆爵士：ride 跳音 + 行走贝斯 + Charleston 和弦 + 弱音小号 |
| 踢踏回声 | 呼叫-应答：复现节奏型 | 120 | 跨步钢琴（stride piano）：低音八度与和弦柱交替的 oom-pah 织体 |
| 剧院三重唱 | 长按：起音 + 收音双判定 | 96 | 慢速轻摇摆 + 三和声人声（双振荡器 + 低通 + LFO 颤音） |
| 橡皮羽球 | 往返对拉：2 拍看球 / 4 拍数拍 | 120 | 轻摇摆四件套，给回球音色留前景 |

### 节目一：橡皮管拳击 RUBBER HOSE BOXING

杂物（馅饼 / 铁砧 / 皮靴）滞空恰好 2 拍飞抵命中点，到点一瞬挥拳。
38 个判定点，难度从每 4 拍一个递进到摇摆反拍双连；
结尾 72–76 拍是 "shave and a haircut, two bits" 终止句，最后两击落在 "two bits" 上。
掷出瞬间有滑笛（slide whistle）上滑提示音；完美命中帽子弹跳 + 星爆。

### 节目二：踢踏回声 ECHO TAPS

呼叫-应答（call & response）结构：搭档在 4 拍内踏出一段舞步（亮踏点 + 踏步声），
下一个 4 拍你按相同的相对节拍原样踏回去——考节奏记忆而非反应。
9 段舞步从 2 踏递进到 6 踏（含摇摆反拍）；舞台上方的节奏卡实时显示
当前舞步的踏点图示（呼叫阶段逐点点亮，应答阶段按判定结果填充/打叉），
聚光灯跟随当班舞者，呼叫末拍有"唔哼！"换班提示音。

### 节目三：剧院三重唱 THE HUMMING TRIO

左边两位歌手按乐句自动开口，你负责三和声的最高声部：
乐句**起拍按住**按键开口、**收拍松开**收声——每个乐句判定起音（onset）与
收音（release）两个时刻，各按统一窗口计分（9 个乐句 = 18 个判定点）。
指挥在乐句起止前一拍各给一声预备提示音（低音=起，高音=收）并抬棒蓄势；
歌词卡显示当前乐句的时值点阵与收声方块。按错时机会独唱走调，邻座歌手冒出"?!"。

```
  乐句时间轴（单个乐句，len = 4 拍）：

   提示音(低)                        提示音(高)
      |                                 |
  ----+----┌────────────────────────┐---+----┐
      ↑    │ 按住 ………………………………………… │       ↓ 松开
    b-1拍  b（起音判定）          b+len-1   b+len（收音判定）
```

### 节目四：橡皮羽球 RUBBER RACKETS

往返对拉（rally），节奏二元化（共 25 次击球）：

- **2 拍常规回合**：弧线低平，球始终可见，看球反应即可，不必数拍；
- **4 拍高吊**：搭档配着滑笛把球挑出画面顶部（滞空 3 拍，顶部留一个
  位置标记），随后两声递升的数拍音"哒、哒"，第三拍正好落点击杀——
  必须数拍子，这是本节目的考点。

漏接时球坠地淡出，搭档换一只新球照常发起下一回合，回合表不会脱轨；
最后一个高吊是终结扣杀，把球抽出场外。

```
  高吊时间轴（拍）：

  你击球      搭档挑高(滑笛↑)   哒        哒        打！
    |            |             |         |         |
    T-4          T-3           T-2       T-1       T
                 └── 球冲出画面顶部，仅剩 ▲ 位置标记 ──┘
```

## 判定规则（四个节目共通）

| 判定 | 时间窗口 | 得分 | 连击 |
| --- | --- | --- | --- |
| 完美 | \|Δt\| ≤ 60 ms | +100 | +1 |
| 不错 | \|Δt\| ≤ 115 ms | +50 | +1 |
| MISS | 超时未击 | 0 | 清零 |

结算评级按加权准确率 `(完美数 + 0.5 × 不错数) / 总数`：
≥ 92% → HIGH LEVEL；≥ 65% → PASS；否则 TRY AGAIN。

```
  按键 (t = now)
    |
    +-- 是否存在 |t - hitTime| ≤ 115ms 的待判定项?
    |     |
    |     +-- 否 --> 挥空/走调（仅反馈，无惩罚）
    |     +-- 是 --> 取 |Δt| 最小者（密集音符按就近消耗）
    |             |
    |             +-- |Δt| ≤ 60ms  --> 完美（+100，连击+1，星爆）
    |             +-- 否           --> 不错（+50，连击+1）
    |
  超时（now > hitTime + 115ms 仍未触发） --> MISS（连击清零）
```

## 视觉：橡皮管风格要素

- **单色赛璐珞调色板**：墨色 / 米白 / 灰调五档，无彩色。
- **角色**：软管四肢（二次曲线外凸模拟无骨节肢体）、派切眼、白手套、
  各色帽子（礼帽 / 草帽 / 高顶帽，完美命中或踢腿时弹跳/歪斜）。
- **背景拟人化与律动**：太阳带脸、树木随拍伸缩、聚光灯随拍呼吸、
  剧院场景的帷幔/脚灯——背景跟着音乐跳舞是该时代动画的标志性手法。
- **胶片做旧**：逐帧颗粒、随机划痕、亮度闪烁、暗角（vignette），开场圈入转场
  （iris-in），菜单/结算做成默片字幕卡（ornate title card）。
- **半调网点（halftone）**：远山、树冠、幕布用平铺点阵图案做暗部，模拟老印刷网纹。

## 实现要点

### 引擎与游戏模块

core.js 持有时钟、调度器、判定窗口、计分、合成器、场景与胶片效果；
游戏模块经 `registerGame({...})` 注册，按约定接口被引擎回调：

```
            ┌────────────────────── core.js ──────────────────────┐
            │ 状态机 menu/playing/result · 输入分发 · 计分/特效     │
            │ AudioContext 时钟 · 前瞻调度器 · 摇摆网格 beatTime()  │
            └──────┬───────────────────────────────────┬──────────┘
       每 0.5 拍回调 │ scheduleStep(beat,t)              │ 每帧回调 draw(now,songBeat)
       每 25ms 回调  │ scheduleAhead(ahead)              │ update(now,songBeat)
       按键/松键     │ onPress(now) / onRelease(now)     │
            ┌──────┴──────┐   ┌──────────────┐   ┌──────┴───────┐
            │ game-boxing │   │   game-tap   │   │ game-chorus  │
            │ 谱面/角色/  │   │ 舞步谱/舞者/ │   │ 乐句表/歌手/ │
            │ 投掷物/配乐 │   │ 节奏卡/配乐  │   │ 指挥/配乐    │
            └─────────────┘   └──────────────┘   └──────────────┘
```

### 单一时钟源 + 前瞻调度

节奏游戏的核心约束是**音画与判定共用一个时间基准**。本项目以
`AudioContext.currentTime`（音频硬件时钟）为唯一时钟源，避免
rAF（requestAnimationFrame，浏览器渲染回调）时间戳与音频时钟之间的漂移。
配乐采用前瞻调度（lookahead scheduling）模式：低精度轮询负责"发现"
即将到期的音符，高精度的 Web Audio 定时队列负责"执行"。

```
  setInterval(25ms) 轮询                      rAF 渲染循环
        |                                         |
        v                                         v
  [前瞻调度器] --提前 120ms 提交--> [Web Audio    [读取 currentTime]
        |                           定时队列]        |
   鼓 / 贝斯 / 人声 / 提示音           |          插值角色动画 / 物体位置
   （采样级精度发声，                  |          （每帧由时间反推位置）
    经摇摆网格 beatTime 映射）         |              /
        \                             |             /
         +--------- AudioContext.currentTime ------+
                       （唯一时间基准）
```

**摇摆量化（swing）**：八分反拍统一推迟到拍内 `SWING` 处（拳击/踢踏/羽球
0.64，三重唱 0.58）。该映射同时作用于配乐调度与谱面判定时刻——反拍音符按摇摆后
的网格判定，听感与判定一致。每个游戏可声明自己的 BPM（beats per minute，
每分钟拍数）与摇摆比，开局时由引擎切换。

物体/动画位置不做增量积分，而是每帧由时间差反推参数位置——渲染掉帧不会
累积误差。暂停直接调用 `AudioContext.suspend()`：音频时钟随之冻结，所有由
它派生的判定与动画在恢复后时序天然一致；退出到菜单时引擎会掐断已预定的
持续人声（killVoices），避免余音拖进菜单。
